# Patterns

Official Patterns document the correct way to compose multiple Core concepts in
Titan to avoid a specific architectural mistake.

A pattern is not documentation — it does not explain how APIs work.
A pattern is not a Recipe — it does not provide importable code.
A pattern addresses one question: how should these Core concepts be composed
together, and what goes wrong when they are not?

→ For the full definition of what makes a pattern official: [internal/design_notes.md](../internal/design_notes.md)
